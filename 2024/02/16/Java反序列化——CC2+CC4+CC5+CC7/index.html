Java反序列化——CC2+CC4+CC5+CC7

Created2024-02-16|Updated2025-08-09|Java
|Word count:505|Reading time:1min|Post View:

前言

前面已经分析了CC1、CC3和CC6，本文把剩下的CC2、CC4、CC5、CC7也学习一下。

CC2

CC2相比于CC1，改变了它的反序列化路口点，并且依赖版本存在区别

  • CC1：
    • 适用于Commons Collections 3.1-3.2.1版本
    • 使用AnnotationInvocationHandler作为反序列化入口点
  • CC2：
    • 适用于Commons Collections 4.0版本
    • 使用PriorityQueue作为反序列化入口点

CC4

CC4和CC2的区别就是CC3和CC1的区别，入口类一致，后半段的执行方法用 InstantiateTransformer 代替了 InvokerTransformer

Transformer类型不同：

  • CC2使用InvokerTransformer直接反射调用方法
  • CC4使用InstantiateTransformer实例化对象，结合TrAXFilter触发TemplatesImpl

执行方式：

  • CC2直接执行命令字符串
  • CC4通过加载恶意字节码执行（更隐蔽）

调用链：

CC5

CC5与CC1相比，核心区别如下：

特性 CC1 CC5
入口类 AnnotationInvocationHandler BadAttributeValueExpException
触发方式 动态代理回调机制 直接toString触发
依赖版本 Commons Collections 3.1-3.2.1 Commons Collections 3.1-4.0
Java兼容性 Java ≤8u71有效 全Java版本有效
利用技术 LazyMap+动态代理 TiedMapEntry+BadAttribute

入口点不同：

  • CC1使用JDK内部的AnnotationInvocationHandler（在Java 8u71后被修复）
  • CC5使用BadAttributeValueExpException（JMX相关类，未被官方修复）

触发机制：

  • CC1依赖动态代理机制和readObject中的entrySet()调用
  • CC5利用BadAttributeValueExpException反序列化时自动调用toString()的特性

CC7

CC7又找了一个新的入口类去触发 LazyMap.get()

入口类：使用 Hashtable作为反序列化入口，比常见的 AnnotationInvocationHandler或 PriorityQueue更隐蔽

触发机制：利用 Hashtable反序列化时对键的 equals()方法调用，进而触发 LazyMap.get()
