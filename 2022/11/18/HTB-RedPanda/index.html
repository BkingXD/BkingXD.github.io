提权root

代码审计

拿到源码，那肯定要进行代码审计，App.java ：

代码大概的流程如下：

1、分行读取 /opt/panda_search/redpanda.log 日志文件(每行一条日志)，isImage() 函数判断是否存在 .jpg 字符(日志中包含一个jpg图片名)。

2、用 parseLog() 函数对日志进行解析，line.split("\\|\\|") 表示分隔符为 || ，分隔成4个字符串转换为Map，有 status_code、ip、user_agent 和 uri 4个键。

3、然后执行 getArtist() 函数，获取 String fullpath = "/opt/panda_search/src/main/resources/static" + uri 这个路径的图片，然后返回这个图片的元数据标签Artist的值(也就是说图片要有Artist这个标签信息)。

4、最后执行 addViewTo() 函数，作用是将解析好的日志写入一个xml文件(实际上，这是一个更新，对xml文件里totalviews和views标签的更新，也就是网页上查看图片的次数，所以这整个jar包运行的作用就是根据日志的变化，更新xml文件)，文件路径为 String xmlPath = "/credits/" + artist + "_creds.xml" 。

如何利用？

从代码中我们知道它会更新xml文件，如果我们修改了xml的内容(写入XXE代码)，就可以利用它在下次更新时触发XXE。我们有xml的路径，但我们查看这个文件夹时，发现我们没有写入的权限，也就无法修改。

这时候就要回到代码本身寻找漏洞，下面这行代码很关键

这行代码就是获取图片路径，它拼接了一个uri，uri是我们请求的资源，也就是 red_panda.log 中最后分隔符后面的部分，例如下面这一条日志，uri=/search

也就是说这个uri的值我们可以控制，而且没有做任何过滤，那就可以让其获取到任意路径的图片，所以图片的信息我们也可以控制，从而使得xml文件的路径也能控制。至此，我们就找到了如何利用XXE的方法。
